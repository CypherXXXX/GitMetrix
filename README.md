## Why GitMetrix?

Developers lack a single place to understand their GitHub activity at a glance вЂ” contribution velocity, language breakdown, streak tracking вЂ” and there's no easy way to have an AI conversation about a codebase without copy-pasting files. GitMetrix solves both problems in one tool.

---

## Features

### Dashboard Analytics
- **Velocity Score** вЂ” Weighted composite score measuring overall development output
- **Active Streak** вЂ” Consecutive days of contributions with streak tracking
- **Total Output** вЂ” Commits, PRs merged, and issues resolved with yearly breakdown
- **Language Breakdown** вЂ” Pie chart distribution of languages across all repos
- **Activity Graph** вЂ” Monthly commit activity visualization using Recharts
- **Top Repositories** вЂ” Sorted by stars, showing language, stars, and forks

### AI Codebase Chat
- **User Repo Selector** вЂ” Browse and select your own repositories to chat with
- **Universal Search** вЂ” Paste any public GitHub URL to index and chat
- **RAG Pipeline** вЂ” Repository files are chunked, embedded (MiniLM-L6-v2), and stored as vectors in Supabase pgvector
- **Semantic Search** вЂ” User queries are embedded and matched against code chunks via cosine similarity
- **Streaming Responses** вЂ” Real-time SSE streaming from Groq (LLaMA 3.3 70B)
- **AI-Generated Suggestions** вЂ” Three repo-specific starter questions generated by analyzing actual file paths and content
- **File References** вЂ” Each response includes clickable file path tags showing which files were referenced

### Authentication & Security
- **GitHub OAuth** via Clerk вЂ” Sign in with GitHub, access profile and repos
- **Protected Routes** вЂ” Dashboard, Chat, and Search require authentication
- **Environment-based Config** вЂ” All secrets stored as environment variables

## How It Works

```
User selects repo в†’ GitHub API fetches file tree в†’ Files are chunked by function boundaries
в†’ Chunks embedded via HuggingFace (MiniLM-L6-v2) в†’ Stored in Supabase pgvector
в†’ User asks question в†’ Query embedded в†’ Cosine similarity search retrieves top 12 chunks
в†’ Chunks + question sent to Groq (LLaMA 3.3 70B) в†’ Response streamed via SSE
```

**Indexing Pipeline (Inngest):**
1. `/api/index` receives a repo URL, creates a DB record, and sends an event to Inngest
2. Inngest function `index-github-repo` runs in the background:
   - Fetches the full repo tree via GitHub API
   - Filters out binaries, lock files, and excluded directories
   - Fetches file content in batches of 5
   - Chunks each file by function/class boundaries with 200-char overlap
   - Generates embeddings in batches of 4 via HuggingFace
   - Stores chunks + embeddings in Supabase `repository_files` table
3. `/api/index/status` polls for completion, then transitions to chat phase

**Chat Pipeline:**
1. User message is embedded via HuggingFace
2. `match_file_chunks` RPC performs pgvector cosine similarity search
3. Top 12 matching chunks are assembled as context
4. Context + conversation history sent to Groq with a detailed system prompt
5. Response streamed back via Server-Sent Events

## Deployment

GitMetrix is designed for **Vercel** deployment. Inngest runs as a serverless function via the `/api/inngest` route вЂ” no separate server or CLI required.

1. Push to GitHub
2. Import into Vercel
3. Add all environment variables
4. Connect Inngest Cloud to your Vercel deployment URL
5. Run the Supabase migration (`supabase/migration.sql`) on your database
